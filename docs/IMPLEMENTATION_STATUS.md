## Cross-implementation validation

Two independent normal-form implementations are compared by the test suite:

1. `src/braid.c` — meet-based left-weighting over permutation braids;
2. `test/python/garside_reference.py` — atom-transfer left-weighting using
   starting/finishing sets, written without reference to the C code path.

`test_braid.py` compares both on random signed words and checks the
left-weighted condition and the `S_8` projection of every result.
`test_cross_implementation.py` drives the full Hash256/XOF pipeline with the
Python normalizer injected and requires byte equality with the default C path.

## Next implementation milestone

1. Phase 0-Sigma instrumentation: rejection counters and larger randomized
   invariant sweeps (Section 14.1 of the specification).
2. Phase 1-Sigma statistics: avalanche, factor/infimum distributions, output
   bias measurements.
3. Reduced-round and Toy-13 instances for Phase 2-Sigma cryptanalysis without
   changing the production APIs.
4. Optional performance work: cache derived constants instead of re-deriving
   them through SHAKE256 on every call.
